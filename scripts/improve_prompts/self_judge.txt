이 프로젝트를 스스로 분석하고, 개선이 가장 시급한 영역을 판단해서 수정해.

판단 기준 (우선순위 순):
1. 실패하는 테스트가 있으면 최우선으로 수정
2. 런타임 에러나 예외 발생 가능성이 있는 코드
3. 테스트 커버리지가 가장 낮은 모듈
4. 성능 병목이 명확한 코드
5. 코드 품질 (리팩토링, 타입 힌트 등)

작업 방식:
- 한 번에 하나의 이슈만 수정
- 수정 후 반드시 uv run pytest tests/ -x -q 실행
- 테스트 통과 후 다음 이슈로 이동
- 변경이 너무 광범위하면 더 작은 단위로 쪼개서 진행
- 절대 기존에 통과하던 테스트를 깨뜨리지 마
- 한 번의 반복에서 10개 이상의 파일을 동시에 수정하지 마

이 과정을 최대한 반복하되, 의미 있는 개선이 더 없다고 판단되면 멈추고
전체 작업 요약을 보고해.
